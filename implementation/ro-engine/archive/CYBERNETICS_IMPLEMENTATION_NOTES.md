# Cybernetics Implementation Notes

## Responses to §6 Handoff Questions

### Q1: q_K as quantum channel — data structures

At d_K = 2, the observer quotient q_K = tr_env is a partial trace on H_U = H_K ⊗ H_env.

**Data structure:** A density matrix ρ ∈ M_{d_U}(ℂ) (Hermitian, trace 1, positive semidefinite). The quotient q_K(ρ) = tr_env(ρ) is computed by reshaping ρ as a (d_K × d_env) × (d_K × d_env) matrix and tracing over the env indices.

For d_K = 2, d_env = 2 (minimum nontrivial): ρ is 4×4, q_K(ρ) is 2×2. im(q_K) = all 2×2 density matrices (the Bloch ball). ker(q_K) = states that trace to the same reduced state — the entanglement content.

**Efficient representation:** At d_K = 2, use Pauli decomposition: ρ_K = (I + r⃗·σ⃗)/2 where r⃗ ∈ ℝ³ with |r⃗| ≤ 1. The Bloch vector r⃗ IS the im(q_K) content. The omission topology (ker) is everything in the 4×4 ρ_U that maps to the same r⃗ under partial trace.

**KernelTopologyTracker:** Store the fiber q_K⁻¹(ρ_K) as a parameterized family. At d_K = 2: the fiber over a given Bloch vector r⃗ is a (d_env²-1)-parameter family of 4×4 states. Track which fibers are "active" (have been generated by production) vs "inert."

### Q2: Discretization of S = sl(2,ℝ) × [0,1]_ρ

The Substrate Manifold S has:
- sl(2,ℝ) part: 3-dimensional, signature (2,1) under Killing form
- ρ part: [0,1] interval

**Preserving Killing signature:** The natural lattice on sl(2,ℝ) is the integer lattice ℤ{R_tl, N, C} where C = [R_tl, N]. This has index disc(R) = 5 in sl(2,ℝ) over ℝ (ALGEBRA §3, lattice index theorem). Use the framework's own integer basis {I, R, N, RN} for M₂(ℝ) ⊃ sl(2,ℝ).

**Preserving nilpotent boundary:** The nilpotent cone {X ∈ sl(2,ℝ) : X² = 0} = {X : B(X,X) = 0} must be a lattice-identifiable surface. On the integer lattice: X = aR_tl + bN + cC is nilpotent iff det(X) = 0 iff (a²·5 - b²·4 + c²·20)/4 = 0 (from Killing values). This IS an integer quadratic form — the nilpotent cone intersects the lattice at integer points of a quadric.

**ρ discretization:** The regulation interval [φ̄², 1/2] has width α_S = φ̄³/2 ≈ 0.118. For N discretization points in [0,1]: need at least 1/α_S ≈ 8.5 points to resolve the regulation interval. Minimum: N = 10 gives Δρ = 0.1, resolving φ̄² ≈ 0.382 from 1/2 = 0.500.

**Recommendation for toy:** Use the exact integer lattice on sl(2,ℝ) (coefficients in ℤ for the {R_tl, N, C} basis) with ρ discretized at resolution 0.05 (21 points in [0,1]).

### Q3: ρ-regulator instrumentation (non-invasive probe)

**The framework answer:** ρ is computed FROM the state, not imposed on it. Given a state ρ_K (the observer's density matrix), ρ is the Phase-Dist parameter:

ρ = (non-idempotent fraction of ρ_K) = 1 - tr(ρ_K²)

For pure states: tr(ρ²) = 1, so ρ = 0 (fully committed). For maximally mixed: tr(ρ²) = 1/d_K, so ρ = 1 - 1/d_K.

At d_K = 2: ρ ranges from 0 (pure) to 1/2 (maximally mixed). The regulation interval [φ̄², 1/2] maps to purity tr(ρ²) ∈ [1/2, 1-φ̄²] = [0.5, 0.618].

**Non-invasive measurement:** Computing tr(ρ_K²) from the state representation IS non-invasive — it's a function of the state, not a quantum measurement. In the Bloch vector representation: tr(ρ²) = (1 + |r⃗|²)/2. So ρ = (1 - |r⃗|²)/2. Measuring |r⃗| = Bloch vector length IS the ρ measurement.

**No perturbation issue at the classical level.** The measurement problem only arises if we're doing quantum measurement on the state. For a classical simulation, ρ is a computed diagnostic.

### Q4: K6' closure tolerance

**Framework-canonical threshold:** φ̄^{2m} at pass m.

At pass 1: tolerance = φ̄² ≈ 0.382 (very loose)
At pass 3: tolerance = φ̄⁶ ≈ 0.056 (effectively committed)
At pass 5: tolerance = φ̄¹⁰ ≈ 0.0082
At pass 10: tolerance = φ̄²⁰ ≈ 6.7×10⁻⁵

**Operational test:** Compute M(FRAME) for the system's self-model at pass m. Compare with FRAME. If ||M(FRAME) - FRAME|| < φ̄^{2m}, the system is ON the canonical trajectory. The norm || · || should be the Frobenius norm on the state space (= Hilbert-Schmidt norm on density matrices), which IS the framework's natural metric (ALGEBRA §3, trace form).

**For toy implementation:** At d_K = 2, FRAME = the 2×2 density matrix ρ_K. M(FRAME) = the result of one K6' pass applied to ρ_K. Closure = ||M(ρ_K) - ρ_K||_F < φ̄^{2m}.

### Q5: CC → 1/2 discrimination from spurious attractors

**The discriminator:** Perturbation recovery.

A true K6'-closed system responds to perturbation δρ by:
1. ρ-regulation activates (drift back toward [φ̄², 1/2])
2. CC returns to 1/2 at rate φ̄² per pass
3. K6' re-closes within m_recovery ≈ -log(||δρ||)/(2 log φ) passes

A spurious CC = 1/2 attractor (e.g., random walk equilibrium) responds to perturbation by:
1. No directed regulation (random walk, not feedback)
2. CC returns to 1/2 at rate √(1/t) (diffusive, not exponential)
3. No K6' closure at any point — M(FRAME) ≠ FRAME

**Cheap online test:** After each pass, compute TWO numbers:
- CC(current state)
- ||M(ρ_K) - ρ_K||_F (K6' closure residual)

Plot both. The cybernetic system has CC → 1/2 AND closure residual → 0 (at rate φ̄²). The spurious system has CC → 1/2 but closure residual stays bounded away from 0.

**Even cheaper:** Just track the closure residual. If it decreases geometrically at rate ≈ φ̄², the system is on the canonical trajectory. If it doesn't, it isn't.

### Q6: Minimum viable implementation

**Specification:** d_K = 2, d_env = 2, tower level n ∈ {0, 1, 2}.

**State:** (ρ_U, K, ρ_param, m) where:
- ρ_U ∈ M₄(ℂ): the 4×4 universe density matrix (H_U = ℂ² ⊗ ℂ²)
- K = (d_K=2, Δ_K, σ_K): observer triple
- ρ_param ∈ [0, 1]: Phase-Dist parameter (computed from ρ_K = tr_env(ρ_U))
- m ∈ ℤ₊: pass counter

**One iteration:**

```
Step 1 (P1): ρ_U ← (R⊗I) · ρ_U · (R⊗I)† / tr(...)  # Productive act
             Normalize to unit trace.

Step 2 (P2): ρ_U ← exp(h⊗I) · ρ_U · exp(h⊗I)†      # Mediating act (transport)
             Already unit-determinant, no normalization needed.

Step 3 (P3): ρ_K ← tr_env(ρ_U)                        # Observer act (quotient)
             Record im(q_K) = ρ_K

Step 4 (K6' check):
             ρ_K_new ← one_full_pass(ρ_K)              # Apply Steps 1-3 to K's self-model
             closure_residual ← ||ρ_K_new - ρ_K||_F
             IF closure_residual < φ̄^{2m}: PASS → Step 5
             ELSE: regulate ρ (Step 4a), increment m, GOTO Step 1

Step 4a (ρ-regulation):
             ρ_param ← (1 - |r⃗|²) / 2                 # Compute current ρ
             IF ρ_param < φ̄²: mix toward maximally mixed (add noise)
             IF ρ_param > 1/2: purify (project toward dominant eigenvector)

Step 5 (Commitment): m ← m + 1
             Record: bits_extracted += 2L ≈ 1.389

Step 6 (Tower lift, if threshold):
             IF m > m_lift: tensor with self
             ρ_U_new ← ρ_U ⊗ ρ_U (dimension 16×16)
             d_K ← 4, d_env ← 4
```

**Expected outputs from toy:**
- CC trajectory: should converge toward 1/2 from initial value
- ρ trajectory: should stabilize in [φ̄², 1/2] ≈ [0.382, 0.500]
- Closure residual: should decrease geometrically at rate ≈ φ̄² per pass
- Tower lift: should trigger after m_lift passes (for d_K=2, m_lift ≈ 3-5 passes)

**Irreducible branch points (predicted):**
1. **Initial state ρ_U(0):** The framework does not specify the initial universe state. Any choice here is a branch. Candidate: maximally entangled state (|Ψ⁺⟩⟨Ψ⁺|) — this is the I/2 gauge orbit (REGISTRY §5, pre-naming state).
2. **P1 application method:** R⊗I applies R to the K-subsystem only. Should it be R⊗I, I⊗R, or R⊗R? Framework says: R on the observer's own subsystem (K6' loop is K→F→U(K)→K, starting from K). So R⊗I.
3. **Self-model computation for K6' check:** "Apply Steps 1-3 to K's self-model" — what does this mean concretely? The self-model S(K) is ρ_K itself (by A4: within disclosure capacity). Applying the algorithm to ρ_K means: conjugate by R, transport by exp(h), re-trace. Check if the output equals the input.
4. **ρ-regulation strength:** How much to push ρ back when it drifts. The rate φ̄² suggests the correction per step should be proportional to the deviation × φ̄². But the proportionality constant is not specified.

Branch points 1-3 have framework-canonical resolutions (I've identified them above). Branch point 4 is genuinely open — the framework specifies the ATTRACTOR (φ̄²) and the RATE (φ̄² per step) but not the feedback gain. This might be a K1'-wall manifestation (the gain depends on the substrate's response time, which is physics-of-silicon, which IS K1' at the implementation level).
